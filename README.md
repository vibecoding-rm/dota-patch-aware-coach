# Dota 2

Proyecto de investigacion de producto para detectar necesidades reales de la
comunidad de Dota 2 y convertirlas en una oportunidad monetizable.

Punto de entrada obligatorio: lee `AGENTS.md` antes de investigar o construir.

## Estructura

- `AGENTS.md`: instrucciones del agente investigador.
- `investigacion/`: planes, preguntas, entrevistas y analisis.
- `evidencia/`: enlaces, notas fechadas y pruebas de demanda.
- `producto/`: oportunidades, MVP, pricing y roadmap.

## Documento principal

- `producto/vision_y_escalado.md`: que queremos construir, por que, como usa IA
  y como se escala desde validacion manual hasta producto pagado.
- `producto/codex_setup.md`: skills, MCP y setup de Codex aplicado al proyecto.

## Regla de trabajo

Primero se valida el problema. Despues se decide que construir.
