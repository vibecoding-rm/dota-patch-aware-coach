# Mejoras propuestas - 2026-06-16

## Diagnostico actual

El proyecto ya paso de demo a MVP tecnico inicial:

- Draft: motor determinista con tests.
- Replay: OpenDota real + reporte determinista + redaccion IA opcional.
- UI: modos principales separados y workspace de coach.
- Calidad: `npm run test`, `npm run typecheck` y `npm run lint` pasan.

La brecha principal ya no es "conectar datos reales de partidas". La brecha
principal es frescura y profundidad de conocimiento de Dota:

- Patch Coach sigue usando cambios mock.
- Draft usa 26 heroes escritos a mano.
- No hay pipeline de patch notes.
- No hay modulo real de vision/wards.
- No hay validacion comercial registrada con usuarios/coaches.

## Respuesta a la pregunta: se pueden hacer trucos, wards y cambios por parche?

Si, pero con limites claros.

Se puede hacer:

- Trucos educativos fuera de partida: creep aggro, pull/stack timings, catapult
  push, warding antes de runas, jugar alrededor de Roshan, item timings.
- Consejos por parche: "este cambio vuelve mejor/peor este timing", siempre con
  fuente, version y confianza.
- Planes de wards: defensivo/agresivo/Roshan/runa/mid/triangulo, dados como guia
  manual o post-game.
- Revision de replay: detectar si el jugador puso wards tarde, no protegio rune,
  no convirtio pelea en torre, murio sin buyback, etc., cuando los datos existan.

No se debe hacer:

- Overlay que lea memoria o estado del cliente.
- Automatizar compras, picks, pings o inputs.
- Mostrar informacion oculta o en vivo que el jugador no deberia tener.
- Prometer ventaja competitiva o subir MMR garantizado.

## Mejoras priorizadas

### 1. Patch Coach real

Objetivo: reemplazar `PATCH_NOTES_BY_HERO` mock por datos versionados.

MVP:

- Tabla/JSON `patches`.
- Tabla/JSON `patch_changes`.
- Ingesta manual asistida de patch 7.41d.
- Campo por cambio: fuente, tipo, target, resumen, impacto, confianza.
- UI que muestre "hecho" vs "inferencia".

Criterio de exito:

- Para 20 heroes populares, la app responde "que cambio para mi pool esta
  semana" sin inventar datos.

### 2. Vision Coach

Objetivo: responder a una necesidad clara: donde wardear y por que.

MVP:

- 12 escenarios de vision:
  - mid runes;
  - proteger carry en safe lane;
  - jugar agresivo en triangulo enemigo;
  - defender triangulo propio;
  - Roshan antes de minuto 20;
  - Roshan late;
  - jugar contra Slark/Riki/Bounty;
  - high ground push;
  - deward cuando vas perdiendo;
  - smoke desde base;
  - wisdom rune;
  - lotus/rune control.
- Cada escenario debe tener:
  - objetivo;
  - quien coloca ward;
  - momento;
  - riesgo;
  - sentry recomendada;
  - como no regalar el ward.

Criterio de exito:

- Un support 4/5 entiende que ward colocar antes de moverse, sin mapa en vivo.

### 3. Mi hero pool semanal

Objetivo: convertir meta actual en decisiones personales.

MVP:

- Usuario elige 5-8 heroes.
- App clasifica:
  - jugar ahora;
  - jugar solo si draft acompana;
  - pausar esta semana;
  - aprender como backup.
- Cruza:
  - comfort;
  - rol;
  - dificultad;
  - patch value;
  - meta externa con atribucion;
  - bracket.

Criterio de exito:

- El usuario sale con 3 heroes y una razon concreta, no con una tier list.

### 4. Reporte post-partida por rol v2

Objetivo: pasar de heuristicas generales a diagnosticos por rol.

MVP:

- Carry: CS min 10, item timings, muertes antes de BKB, conversion a Roshan.
- Mid: runas 2/4/6/8, torre mid, rotaciones, control de power rune.
- Offlane: presion de T1, aura/blink timing, crear espacio.
- Support 4: runas, wisdom, ganks, sentries, smoke.
- Support 5: pulls, proteccion carry, vision defensiva, saves.

Criterio de exito:

- El reporte no acusa a un support de "poco dano a torres" ni a un carry de
  "falta de wards" como error principal salvo evidencia clara.

### 5. Validacion comercial antes de mas features grandes

Objetivo: evitar construir mas sin saber si alguien paga.

Prueba de 7 dias:

- 5 reportes reales gratis con feedback escrito.
- 5 reportes pagos a USD 3.
- 2 coaches hispanos probando workspace.
- 1 comunidad Discord probando reporte compartible.

Criterio de exito:

- 3 usuarios piden segundo reporte.
- 1 coach dice que le ahorra tiempo real.
- Al menos 1 persona paga o acepta precio.

## Orden recomendado

1. Corregir textos mojibake en UI/docs visibles.
2. Patch Coach real para 7.41d.
3. Vision Coach con 12 escenarios.
4. Reporte v2 por rol.
5. Validacion con usuarios reales.
6. Solo despues: STRATZ, pagos, cuentas, historial persistente.
